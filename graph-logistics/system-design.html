System Architecture

Runtime topology across five tiers. The site is a fully client-side React SPA: there is no first-party application server — the React provider & route layer runs entirely in the browser and talks directly to two external services. Solid borders = verified from code; dashed = inferred.

Inferred components & notes

  • Vercel Static Host (Zone 2, inferred) — no server code in repo; inferred from vercel.json rewrite /(.*) → / and public/_redirects. The host serves the static Vite bundle and rewrites all paths to the SPA shell.
  • Products REST API (Zone 5, verified URL) — https://midnight-sec-back.onrender.com/api/products/ hardcoded in src/Context.jsx and src/Pages/Admin/AdminPanel.jsx. The backend implementation is external/out-of-repo; its schema is inferred from field access (_id, name, description, image, createdAt).
  • Telegram Bot API (Zone 5, verified) — https://api.telegram.org/bot<token> in src/lib/telegram.js; credentials from import.meta.env.VITE_TELEGRAM_BOT_TOKEN / VITE_TELEGRAM_CHAT_ID. No .env file is committed, so the values are unresolved.
  • Auth / credential store (Zone 3/4, verified) — client-only: ProtectedRoute.jsx and Login.jsx compare localStorage 'token' to btoa('moglobeadmin:adminmoglobe'). No server-side authentication exists; this is decorative access control.
